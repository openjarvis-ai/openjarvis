# OpenClaw Studio

A modern, production-ready web application for **screen recording** and **workflow review** built with Next.js 14, TypeScript, and Tailwind CSS.

---

## Features

### Screen Recording
- Record entire screen, specific window, or browser tab
- Pause / resume / stop controls
- Live recording indicator with timer
- Video preview with playback controls
- Download recording locally as `.webm`
- Upload recording to backend API
- Graceful permission error handling

### Workflow Review & Comments
- Browse workflows generated by OpenClaw and Opus
- View workflow details: title, steps (with status timeline), metadata, tags
- Write text comments about any workflow
- Send comments to Opus via `POST /api/send-to-opus`
- Toast notifications for success / error feedback
- **Only** text comments accepted вЂ” no ratings, scores, or annotations

### Dashboard
- Overview stats: recordings, workflows, comments sent, total duration
- Recent activity feed combining recordings and workflows
- Quick-action button to start a new recording

## Setup Instructions

### Prerequisites
- **Node.js** 18.17+ (LTS recommended)
- **npm** 9+ or **pnpm** 8+

### Install & Run

```bash
# 1. Navigate into the project
cd openclaw-studio

# 2. Install dependencies
npm install

# 3. Start the dev server
npm run dev

# 4. Open in browser
open http://localhost:3000
```

### Build for Production

```bash
npm run build
npm start
```

## API Reference

### `GET /api/workflows`
Returns all mock workflows.

**Response:**
```json
{
  "success": true,
  "message": "Workflows fetched successfully",
  "data": [ /* Workflow[] */ ]
}
```

### `POST /api/send-to-opus`
Send a text comment about a workflow to Opus.

**Request body:**
```json
{
  "workflowId": "wf_001",
  "comment": "The workflow missed the step where I opened the dashboard.",
  "recordingId": "rec_456"  // optional
}
```

**Validation rules:**
- `workflowId` вЂ” required string
- `comment` вЂ” required non-empty string, max 2000 chars
- `recordingId` вЂ” optional string
- Any additional fields are **rejected** (no ratings, scores, etc.)

**Response:**
```json
{
  "success": true,
  "message": "Comment sent to Opus successfully.",
  "commentId": "cmt_abc123",
  "timestamp": "2026-02-15T10:30:00Z"
}
```

### `POST /api/upload-recording`
Upload a recorded video file.

**Request:** `multipart/form-data` with a `file` field.

**Response:**
```json
{
  "success": true,
  "message": "Recording uploaded successfully.",
  "recordingId": "rec_xyz789",
  "size": 48500000,
  "timestamp": "2026-02-15T10:30:00Z"
}
```

## Key Design Decisions

1. **Feature-based folder structure** вЂ” Components are grouped by feature (`recorder/`, `workflows/`, `dashboard/`) for scalability, not by technical type.

2. **Custom `useScreenRecorder` hook** вЂ” All MediaRecorder logic is encapsulated in a single hook with a clean return interface. Components never touch the MediaRecorder API directly.

3. **Context-based toast system** вЂ” Lightweight toast notifications via React Context, no external dependency. Toasts auto-dismiss and support success/error/info variants.

4. **Strict comment-only API** вЂ” The `/api/send-to-opus` route explicitly rejects any payload fields beyond `workflowId`, `comment`, and `recordingId`. This enforces the product requirement that only text comments are sent.

5. **Design tokens via Tailwind** вЂ” Custom `surface-*` and `brand-*` color scales, custom shadows, custom fonts (DM Sans + Instrument Serif + JetBrains Mono) for a premium, non-generic look.

6. **Responsive sidebar** вЂ” Collapsible on desktop (72px vs 260px), slide-out drawer on mobile with backdrop overlay.

7. **Progressive UI states** вЂ” Every data-dependent view has loading skeletons, empty states, and error handling. No blank screens.

## Future Improvements

- **Dark mode toggle** вЂ” The CSS supports `dark:` variants; add a theme switcher in settings
- **Real backend** вЂ” Replace mock APIs with a database (Prisma + PostgreSQL) and object storage (S3) for recordings
- **Authentication** вЂ” Add NextAuth.js for user sessions
- **Workflow search & filtering** вЂ” Add search bar, tag filters, and source filters to the workflow list
- **Recording thumbnails** вЂ” Generate video thumbnails using canvas extraction
- **Real-time recording status** вЂ” WebSocket-based status updates for collaborative review
- **Keyboard shortcuts** вЂ” `Ctrl+Shift+R` to start recording, `Esc` to stop
- **E2E tests** вЂ” Playwright tests for critical flows
- **Framer Motion** вЂ” Replace CSS animations with Framer Motion for richer page transitions and list animations
